📋 项目概况

SCQ 是一个基于 SairFramework 插件框架开发的即时聊天系统，采用 C/S 架构，支持私聊、群聊、好友管理、指令传达和 IR 脚本执行。服务端负责用户管理和消息路由，客户端提供完整的 Swing GUI 交互界面。

🖥️ 服务端

SCServerCore 监听 TCP 端口，为每个连接创建 ClientSession 线程。通过 UserManager 和 GroupManager 管理注册用户与群组数据，数据以 JSON 持久化到本地文件。

💻 客户端

SCClientCore 维持单 Socket 持久连接，发送线程从 BlockingQueue 取消息写入，接收线程循环读取后回调 MessageHandler。聊天记录使用 AES/CBC 加密本地存储。

📡 通信协议

传输格式

采用 长度前缀 + JSON 字符串 的二进制协议：

// 发送端
byte[] jsonBytes = msg.toJson().getBytes(StandardCharsets.UTF_8);
dos.writeInt(jsonBytes.length);     // 4字节长度前缀
dos.write(jsonBytes);               // JSON 字节
dos.flush();

// 接收端
int length = dis.readInt();
byte[] buf = new byte[length];
dis.readFully(buf);
SCMessage msg = SCMessage.fromJson(new String(buf, UTF_8));

消息 JSON 结构

{
  "t": "COMMAND",          // 消息类型
  "f": 1001,                  // 发送方 UID
  "i": 1002,                  // 接收方 UID / 群组 GID
  "d": "Hello World",     // 主要载荷
  "o": "{\"key\":\"val\"}", // 额外数据
  "ts": 1716982400000         // 时间戳
}

// ClientActions.handleUISendCommand
if (trimmed.startsWith("\"") && trimmed.endsWith("\"")) {
    String path = trimmed.substring(1, trimmed.length() - 1);
    String irContent = new String(
        Files.readAllBytes(Paths.get(path)), UTF_8);

接收端

// SCClientActivity.handleCommand
if (command.startsWith("IR:")) {
    String irContent = command.substring(3);
    // IR脚本始终通过IRRunnable模式执行
    SairCons.runner(true, irContent);
}
设计原则：IR 脚本作为程序化内容（machine-to-machine），始终自动执行，不依赖 autoExecute 开关。普通 SairFramework 命令（插件/功能名 参数 格式）则受 scqc/autoExec true 控制。

👥 好友系统 — 双向添加机制

sequenceDiagram participant A as 客户端A
(UID: 1001) participant S as 服务端 participant B as 客户端B
(UID: 1002) A->>S: CONTACT_ADD 添加1002 S->>S: addContact(1001, 1002) S->>S: addContact(1002, 1001) S-->>A: OK + 通讯录推送 S-->>B: CONTACT_ADDED 通知 S-->>B: 通讯录推送 Note over A,B: 双向好友关系建立完成

关键设计

  • 双向添加：A 加 B 时，服务端同时执行 addContact(A, B) 和 addContact(B, A)
  • 实时通知：新增 CONTACT_ADDED 消息类型，被添加方立即收到通知
  • 通讯录刷新：每次好友变更后自动推送完整通讯录给双方
  • UID + 名称显示：通讯录格式为 [在线] UID:1001 - 用户名 - 签名

服务端

  • 主线程阻塞等待新连接
  • 每个客户端一个 ClientSession 线程
  • 同步读一个消息 → 处理 → 返回结果
  • synchronized 保护 DataOutputStream

客户端

  • 发送线程从 BlockingQueue 取消息
  • 接收线程持久循环读取（参照 nsc AutoPrintln）
  • 消息回调在接收线程中执行
  • UI 更新通过 SwingUtilities.invokeLater

🟢 在线状态判断

参照 nsc 项目的 socket-driven 机制，在线状态完全基于实际 TCP 连接状态判断，而非依赖 UserInfo.isOnline 布尔字段：

// SCServerCore.java
public boolean isUserOnline(long uid) {
    ClientSession session = sessionMap.get(uid);
    return session != null && isSessionActive(session);
}

// 断线时自动清理：Socket IOException → logout → unregisterSession

🔐 安全性

传输安全

  • 用户密码 SHA-256 哈希存储，永不明文传输
  • 服务端验证登录时比对哈希值
  • 消息通过 TCP 长连接传输（局域网场景）

存储安全

  • 聊天记录 AES/CBC 模式加密存储客户端
  • 加密密钥由用户密码派生
  • 服务端不存储任何聊天记录，仅存用户/群组信息
  • handleOk 防护：必须含 "passwordHash" 才解析为 UserInfo

🚀 构建与部署

编译命令

:: JDK 1.8 路径
SET JDK=D:\APPs\SairFrameWork\8x64\bin\javac
SET CP=D:\APPs\SairFrameWork\SFW.jar
SET SRC=d:\Users\Desktop\FileExp\scq
SET OUT=d:\Users\Desktop\FileExp\scq\out

:: 编译
%JDK% -encoding UTF-8 -cp %CP% -d %OUT% ^
  %SRC%\sair\scq\*.java ^
  %SRC%\sair\scq\protocol\*.java ^
  %SRC%\sair\scq\model\*.java ^
  %SRC%\sair\scq\server\*.java ^
  %SRC%\sair\scq\client\*.java ^
  %SRC%\sair\scq\ui\*.java ^
  %SRC%\sair\scq\acts\*.java

:: 打包（必须 cfm 包含自定义 MANIFEST.MF）
%JDK:javac=jar% cfm %SRC%\scq.jar %SRC%\META-INF\MANIFEST.MF -C %OUT% .

:: 部署
copy %SRC%\scq.jar D:\APPs\SairFrameWork\plugins\exection\scq.jar

MANIFEST.MF

Manifest-Version: 1.0
ACT: sair.scq.SCServerActivity;sair.scq.SCClientActivity;
⚠️ 注意：必须使用 jar cfm 命令明确指定自定义 MANIFEST.MF，jar cf 会生成默认清单覆盖掉 ACT: 行，导致 SairFramework 无法识别 Activity。

📝 总结

SCQ 插件实现了完整的即时聊天系统，涵盖用户认证、私聊群聊、好友管理、群组权限、指令传达和 IR 脚本执行等核心功能。系统采用标准 C/S 架构，通过 TCP 长连接和自定义 JSON 协议实现高效通信。UI 遵循 SairFramework 设计规范，透明圆角化控件与框架主题深度融合。

历经多轮迭代，修复了管理员数量公式、好友单向添加、在线状态虚判、IR 脚本执行受阻等关键问题，当前版本功能完整、运行稳定。

Core

PersistenceManager

sair.aiagent.core · 678 行

SQLite 统一持久化核心 —— 替代 memory.json / journal.json / history.json / emotion.json / context.json 五个独立 JSON 文件。

单文件数据库 aiagent.db，WAL 模式（PRAGMA journal_mode=WAL），synchronous=NORMAL，busy_timeout=3000ms。
5 张表：memories（记忆条目 + category/importance）、memories_fts（FTS5 全文索引）、 journal（会话日志，最多 80 条）、conversations（对话历史 + token 估算）、 app_state（键值状态，存情绪/上下文摘要）。

FTS5 触发器：INSERT/UPDATE/DELETE 自动同步全文索引，搜索失败自动回退 LIKE 查询。
旧数据迁移：首次初始化自动将 5 个 JSON 文件导入 SQLite 并重命名为 .bak 备份。
事务安全：所有写操作在 synchronized(lock) 保护下执行，WAL checkpoint 在 close() 时完成。

Core

AgentExecutor

sair.aiagent.core · 1080 行

项目最大文件，ReAct 推理-行动循环引擎。

核心流程：execute(task) → runLoop()（最多 30 轮）→ 每轮 buildSystemPrompt() 融合 6 种上下文 → DeepSeekClient.chatStream() 流式推理 → 正则 parseActions() 匹配 XML 标签 → executeAction() 执行操作 → 捕获结果注入下轮 system prompt。

execq 通道：executeQq(task, systemPrompt, memMgr) 专为 OneBot QQ 设计， 受限标签白名单（cmd/web/readdir），自动绕过确认门控，独立上下文。 executeCmdExecq() 在执行 <cmd> 前校验插件白名单（extractPluginName() 提取插件名），空白名单=全部拒绝。

上下文注入链：静态提示词（缓存） + 上轮 Agent 总结 + Chat 历史摘要 + 会话日志（JournalManager） + 情绪状态（EmotionManager） + 记忆检索（MemoryManager）。

11 种 XML 标签：cmd / readfile / readdir / sys / evaljs / eval / web / remember / download / superise / editprompt。

安全机制：SSRF 内网地址黑名单，高危操作经 ConfirmationGate 确认， 每轮后清除线程中断标记防止污染。

EmotionManager

sair.aiagent.core · 301 行

AI 情绪状态机，模拟类人情感。

情绪词典：辱骂词 17 个（如"笨蛋""废物""蠢"）→ -20 happiness； 夸赞词 18 个（如"真棒""厉害""爱你"）→ +15 happiness； 安慰词 18 个（如"别难过""加油""抱抱"）→ +25 happiness。

阈值机制：happiness ≤ -30 或连续失败 ≥ 3 次 → Agent 暂停（CountDownLatch 阻塞， 等待用户安慰/指导，120 秒超时自动恢复）；happiness ≥ 80 且连续夸赞 ≥ 2 次 → 触发彩蛋。

持久化：状态序列化到 app_state("emotion")（SQLite，Gson JSON）， 高频操作仅标记脏数据，由 flushSave() 在 Agent 循环结束时统一写入，减少磁盘 I/O。
支持 buildEmotionContext() 生成注入 system prompt 的情绪描述文本。
性别默认为 female，影响 AI 的语气和表达风格。

StreamPrinter

sair.aiagent.core · 327 行

单例流式输出核心（getInstance() 全局唯一）。
内部维护 LinkedBlockingQueue<Item>，支持两种输出通道： 行输出（系统消息，offerLine() / offerPlain()，无延迟直出）和 字符输出（AI 流式，offer(char)，10ms 延迟打字机效果）。
start() 先全速排空旧队列再启动新流；守护线程永续运行无需手动启停。
SFW 换行适配：\n 自动转换为 \r\n。
预定义颜色：C_AI（翠绿 100,255,180）和 C_THINK（淡紫蓝 180,180,255）。

核心打印循环
private void printLoop() {
    while (true) {
        Item item = queue.take();  // 守护线程，永久阻塞等待
        if (item.text != null) {
            // 行输出（系统消息）—— 无延迟，直接通过EdtUtils
            SairCons.print(item.color, item.text + (item.newline ? "\r\n" : ""));
        } else {
            // 字符输出（AI 流式）—— 10ms 延迟
            if (item.ch == '\n') SairCons.print(item.color, "\r\n");
            else if (item.ch != '\r') SairCons.print(item.color, String.valueOf(item.ch));
            Thread.sleep(10);
        }
    }
}

SysConsoleExecutor

sair.aiagent.core · 290 行

跨平台系统命令执行器。ProcessBuilder 封装，自动识别操作系统切换 cmd.exe /c（Windows）或 /bin/sh -c（Linux/macOS）。

JournalManager

sair.aiagent.core · 63 行

追加式会话日志管理器 —— SQLite 门面。内部完全委托 PersistenceManager 的 journal 表。
每条日志包含：时间戳、角色（user/assistant/agent）、模式（chat/exec/execs）、内容摘要（截断 300 字符）、 执行结果（截断 500 字符）。
容量控制：最多保留 80 条，超出时自动移除最旧条目（SQL DELETE WHERE id NOT IN ... LIMIT 80）。
buildRecentContext() 取最近 25 条日志构建上下文文本（上限 5000 字符）， 注入 Agent system prompt，实现跨重启的长期记忆。
Chat 模式下也记录用户消息和 AI 回复，形成完整对话日志。

ConversationHistory

sair.aiagent.core · 173 行

对话历史管理器 —— 滑动窗口 + SQLite 持久化。
维护与 AI 的完整对话上下文，通过 token 估算实现最多约 900,000 token 的上下文窗口（约 1M 的 90%）。
当累计 token 超限时，自动移除最旧消息，但至少保留最近 4 条消息（2 轮对话）作为保底。
Token 估算采用经验公式：中文≈0.6 token/字，英文≈0.25 token/字。
持久化：设置 PersistenceManager 后每次 add/clear 自动保存到 SQLite conversations 表，SFW 重启后恢复。

ConfirmationGate

sair.aiagent.core · 150 行

统一高危操作确认门控。CountDownLatch(1) 实现同步阻塞等待。
await(type, desc, timeoutMs)：打印确认提示 → 阻塞等待 → 超时返回 false（默认 60 秒）。 confirm(boolean)：释放 latch，返回确认结果。
setBypassConfirm(true)：跳过等待，直接返回 true（execs 自由模式）。
所有状态赋值（latch / result / pendingType / pendingDesc）在 synchronized 块内完成， 超时后 latch 置 null 防止 isAwaiting() 误判。

OneBot

OneBotServer

sair.aiagent.onebot · 350 行

OneBot v11 反向 WebSocket 服务器，RFC 6455 协议手动实现，零第三方依赖，JDK 8 兼容。

WebSocket 握手：解析 HTTP Upgrade 请求头，SHA-1 + Base64 计算 Sec-WebSocket-Accept， 逐字节读取避免 BufferedReader 预读吞帧。
帧解析：支持 TEXT / PING / PONG / CLOSE 帧类型，自动处理掩码解码， 支持分片帧重组（continuation frame）。
消息发送：sendPrivateMsg() / sendGroupMsg() 构建 OneBot v11 API 调用 JSON，通过 WebSocket 帧发送。
鉴权：支持 Access Token 验证（比较 HTTP 头 Authorization: Bearer xxx）。 start() / stop() 管理 ServerSocket + 客户端连接线程生命周期。

OneBot

QQMessageHandler

sair.aiagent.onebot · 553 行

QQ 消息处理核心 —— 消息解析、过滤、路由到 AI Agent。

无依赖 JSON 解析：extractString() / extractLong() / extractObject() / extractArray() 完整的手动 JSON 解析器， 支持字符串转义、嵌套对象/数组、Unicode 转义。
消息过滤：群消息仅响应 @机器人 或引用回复，通过 checkAtAndReply() 分析消息段（at / reply / text），selfId=0 时不做精确匹配避免误判。
execq 通道：executeQqAgent() 专为 QQ 设计的受限 Agent 通道 —— 独立提示词（通过 AiConfig.getExecqPrompt() 配置）、独立记忆（QQMemoryManager 按 QQ 号隔离）、 独立上下文（每个 QQ 的对话历史独立存储）。仅开放 cmd/web/readdir 标签，自动绕过确认门控。
防重复：processingMessages 集合实现消息级去重， remove 操作在异步任务 finally 块中执行，防止竞态。
线程池：3 线程固定线程池处理 QQ 消息，避免阻塞 WebSocket 读取线程。

CLI Reference

用户命令参考

所有命令以 ai/ 为前缀，通过 SFW 控制台输入。共 33 个用户命令（含 16 个 OneBot QQ 机器人命令）。AiAgent 主线 15 个 XML 标签，QQ execq 通道 16 个标签（10 种群管标签 + 6 种工具标签）。v2.3 更新：多模态图片识别（DeepSeek V4 Vision API），QQ 消息中的图片自动传入 AI 进行视觉分析。

命令 参数 分类 说明
ai/setkey [API Key] 配置 设置 DeepSeek API 密钥，AES 加密存储，自动持久化到 config.properties
ai/seturl [URL] 配置 设置 API 地址，默认 https://api.deepseek.com，带 URL 格式校验
ai/setmodel [模型名] 配置 设置模型，默认 deepseek-chat。旧值自动升级到新默认值
ai/setprompt [提示词] 配置 设置系统提示词（追加到硬编码默认后，持久化到 config.properties）。AI 也可通过 <editprompt> 自行修改
ai/showprompt — 配置 查看当前系统提示词全文
ai/info — 配置 查看配置摘要（Model / URL / Key状态）+ 对话统计 + 数据目录路径（aiagent.db）
ai/chat [消息] 对话 与 AI 对话，流式打字机输出。自动注入 Agent 执行结果作为上下文。触发情绪检测
ai/exec [任务] Agent Agent 安全模式。高危操作需 ai/yes ai/no 确认（60s超时自动拒绝）。注入：Chat历史 + 上轮Agent总结 + 日志 + 情绪
ai/execs [任务] Agent Agent 自由模式。绕过所有确认，自动执行。注入的上下文与 exec 相同
ai/reset — 对话 清空对话上下文（ConversationHistory）+ Agent 记忆链 + 清空磁盘缓存
ai/stop — 对话 立即中断当前流式输出：中断 API 线程 → 全速排空打印机 → 丢弃剩余队列
ai/mood — 情绪 查询 AI 当前心情（emoji + 描述）+ 开心值 + 连续失败/夸赞次数 + 性别
ai/memories — 记忆 列出所有已存储的持久化记忆（SQLite aiagent.db）
ai/forget [ID] 记忆 按 ID 删除指定记忆
ai/forgetall — 记忆 清空所有持久化记忆
ai/yes — 安全 确认 AI 请求的高危操作（通过 ConfirmationGate 释放 latch，Agent 继续执行）
ai/no — 安全 拒绝 AI 请求的高危操作（latch 释放但返回 false，Agent 跳过此操作）
QQ Bot

OneBot v11 QQ 机器人命令

通过反向 WebSocket 连接 QQ，execq 独立通道：独立提示词、统一记忆库（所有对话历史统一管理）、插件白名单门控。共 16 个 OneBot 命令。

v2.1 新增：主人消息添加 execs: 前缀可触发完整 execs 链路，静默执行所有操作，实时推送每轮思考结果给主人。支持 <stop> 标签紧急停止。群管理中 AI 可执行 10 种群管标签（<ban> 禁言/<kick> 踢出/<muteall> 全禁/<setadmin> 设管/<setcard> 改名/<setgroupname> 改群/<leavegroup> 退群/<block> 拉黑/<unblock> 解黑/<delfriend> 删友）。提示词完全分层隔离（主线 15 标签 vs OneBot 16 标签）。

命令 参数 分类 说明
ai/onebotconnect — 连接 启动 OneBot 反向 WebSocket 服务，等待 QQ 端连接
ai/onebotdisconnect — 连接 停止 OneBot 服务，断开所有 QQ 连接
ai/onebotstatus — 连接 查看 OneBot 服务运行状态（监听端口、连接数等）
ai/onebotsetport [端口号] 配置 设置 WebSocket 监听端口，默认 5800
ai/onebotsettoken [Token] 配置 设置 Access Token 用于鉴权（与 QQ 端保持一致）
ai/onebotsetselfid [QQ号] 配置 设置机器人自身 QQ 号，用于群消息 @ 检测与过滤
ai/onebotsetprompt [提示词] 配置 设置 execq 通道独立系统提示词（追加到硬编码默认后，持久化到 config.properties）
ai/onebotshowprompt — 配置 显示当前 execq 通道的系统提示词全文
ai/onebotwhitelist — 安全 显示 execq <cmd> 插件白名单及配置文件路径（config.properties）
ai/onebotwhitelistadd [插件名] 安全 添加插件到 execq 白名单（如 Filer），自动保存配置并立即生效
ai/onebotwhitelistremove [插件名] 安全 从 execq 白名单移除插件，自动保存配置并立即生效
ai/onebotenableproactive — 功能 启用主动查看功能：AI 每 5 分钟自动检查群聊并参与讨论
ai/onebotdisableproactive — 功能 禁用主动查看功能
ai/onebotaddgroup [群号] 功能 添加监听的群号到主动查看列表
ai/onebotremovegroup [群号] 功能 从主动查看列表中移除群号
ai/onebotlistgroups — 功能 列出所有已添加监听的群号

⚙️ execq 通道机制

execq 是专门为 QQ 消息设计的独立 Agent 通道，完全隔离于主 AiAgent 通道：

独立提示词 — 通过 ai/onebotsetprompt 独立配置，不影响主通道
统一记忆库 — 使用 UnifiedQQMemoryManager，所有对话存储在 unified_memory.db，通过 source_type/source_id 标记区分群聊/私聊
受限标签 — execq 共 16 个标签：<stop> <split> <cmd> <web> <readdir> <setname>（工具类，自动绕过确认）
群管标签 — <ban> <kick> <muteall> <setadmin> <setcard> <setgroupname> <leavegroup>（群管类，仅群聊生效）
用户管理 — <block> <unblock> <delfriend>（用户管理类，OneBot 层拦截） 插件白名单 — <cmd> 受插件级白名单门控，空白名单=全部拒绝，通过 ai/onebotwhitelist 系列命令管理
群管理能力 — AI 可通过 NapCatApi 执行警告/禁言/踢出/拉黑操作
群消息过滤 — 仅响应 @机器人 或提到机器人名字或引用回复的消息
好感度决策 — 7步优先级链处理群邀请：主人→反滥用→好感度→背叛者→阈值判断
<stop> 检测 — 主人可通过发送含 <stop> 的消息直接停止 Agent 执行

Internal API

Agent XML 操作标签

AI Agent 通过 XML 标签发出操作指令，由 AgentExecutor.parseActions() 正则解析。共 15 种标签。

标签 格式 需确认 说明
<cmd> <cmd>plugin/func args</cmd> 是 执行 SFW 插件命令（如 Filer/read、IDE/open）
<sys> <sys>dir /s</sys> 是 执行系统命令（Windows cmd / Linux sh），30 秒超时，UTF-8 输出
<readfile> <readfile>path/to/file</readfile> 是 读取文本文件，UTF-8/GBK 自动识别，上限 10MB / 50000 字符
<readdir> <readdir>path/to/dir</readdir> 是 递归列出目录结构，目录优先排序
<evaljs> <evaljs>js code</evaljs> 是 Nashorn 引擎即时执行 JavaScript，返回值自动转字符串
<eval> <eval>java code</eval> 是 JavaCompiler 即时编译执行。须定义 public void run() 方法。一次性：编译→加载→执行→GC
<web> <web>https://...</web> 是 HTTP GET 获取网页内容。SSRF 黑名单：拒绝 localhost/127.0.0.1/内网地址
<download> <download>https://...</download> 是 下载文件到 dataDir/downloads/，自动创建目录
<remember> <remember>重要信息</remember> 否 记录重要信息到持久化记忆（MemoryManager），后续自动检索注入
<superise> <superise>🎉 太棒了！</superise> 否 弹出彩蛋/撒娇窗口。支持 emoji + 文案。粉色（撒娇）/ 橘色（惊喜），3-8 秒自动关闭
<sendimage> <sendimage>文字/路径/URL</sendimage> 否 AWT/Graphics2D 将文字渲染为 PNG 图片发送，或直接发送已有图片文件/URL。QQ 通道发送到聊天，控制台模式保存到 dataDir/rendered/
<sendrecord> <sendrecord>路径或URL</sendrecord> 否 发送语音/音频消息。支持本地文件路径或网络 URL，格式 .amr/.silk/.mp3
<sendfile> <sendfile>本地路径</sendfile> 否 发送本地文件到 QQ 聊天。⚠ 仅主人（masterQQ）可用，非主人调用将被拒绝
<stop> <stop></stop> 否 立即停止 Agent 执行循环。AI 输出此标签后 AgentExecutor 调用 markStopped() 中断。QQ 消息中主人可直接发送 <stop> 停止 Agent。三通道覆盖：QQ直接检测 + execq AI输出 + 控制台TAG_PATTERN
<editprompt> <editprompt>新提示词</editprompt> 否 AI 修改自身 system prompt。要求 ≥ 30 字符，实现性格自我演化
Deployment

部署指南

编译、打包、部署三步完成。

1

编译

使用 SFW 内置 JDK (8x64) 编译全部 34 个源文件。 classpath 需包含 SFW.jar 和 sqlite-jdbc-3.42.0.0.jar。

PowerShell
javac -encoding UTF-8 \
  -cp "SFW.jar;plugins/bootlib/sqlite-jdbc-3.42.0.0.jar" \
  -d build \
  sair/aiagent/**/*.java
2

打包

使用 jar 命令打包，MANIFEST.MF 必须包含 ACT: sair.aiagent.AiAgentActivity;

PowerShell
jar cfm AiAgent.jar \
  META-INF/MANIFEST.MF \
  -C build .
3

部署

将 AiAgent.jar 复制到 SFW 的 plugins/exection/ 目录， 启动 SFW 后即可使用 ai/ 命令。

PowerShell
Copy-Item AiAgent.jar \
  plugins/exection/AiAgent.jar
Quick Start

快速开始

部署后在 SFW 控制台输入以下命令开始使用。

SFW Console
# 1. 设置 API Key
ai/setkey sk-your-deepseek-api-key

# 2. 与 AI 对话（自动感知 Agent 历史 + 情绪状态）
ai/chat 你好，请介绍一下你自己

# 3. Agent 安全模式 —— 高危操作需确认（ai/yes）
ai/exec 列出所有已加载的插件

# 4. Agent 自由模式 —— 无需确认，自动执行
ai/execs 分析插件功能并分类

# 5. Chat 自动感知 Agent —— 询问之前做了什么
ai/chat 刚才exec分析了哪些插件？

# 6. 查询 AI 心情
ai/mood

# 7. 查看配置和统计
ai/info
ai/memories

# === OneBot QQ 机器人 ===
# 8. 配置并启动QQ机器人
ai/onebotsetport 5800
ai/onebotsettoken my-secret-token
ai/onebotsetselfid 123456789
ai/onebotconnect

# 9. 设置execq提示词（可选）
ai/onebotsetprompt 你是Sair的QQ助手，请用中文友好回复

# 10. 查看状态
ai/onebotstatus
ai/onebotshowprompt

# 11. 管理插件白名单（execq安全门控）
ai/onebotwhitelist
ai/onebotwhitelistadd Filer
ai/onebotwhitelistadd IDE

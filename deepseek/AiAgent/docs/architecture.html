核心机制

单例守护线程

StreamPrinter 通过 getInstance() 返回全局唯一实例。内部 daemon 线程 在首次实例化时启动，永续运行，无需手动启停。所有 Chat 和 Agent 模式共享同一输出队列， 无竞态覆盖风险。

双通道输出

行输出（系统消息）：通过 offerLine() 入队，消费者 无延迟直出。
字符输出（AI流式）：通过 offer(char) 入队，消费者 10ms 延迟逐字打印。
两种类型通过 LinkedBlockingQueue 严格保证 FIFO 顺序。

流式排空策略

每次 start() 调用时：
① 生产者线程先 offerLine("…") 放入排空标记
② 消费者检测到标记后，全速从队列取出剩余元素（无延迟）
③ 排空完成后，设置新流颜色，恢复正常 10ms 延迟输出
此机制确保新流到来时旧内容完整输出，不截断不丢失。

EDT 线程安全

SFW 控制台基于 Swing JTextPane，仅 EDT 线程可安全写入。
StreamPrinter 的 daemon 线程不在 EDT 中运行，所有输出均通过 EdtUtils 路由，内部统一调用 SairCons.print()（已在 EDT 安全包装内）。

API 线程（生产者） → 阻塞队列 → 打印 daemon 线程（消费者） → SairCons → 控制台

队列空 + 生产者结束 → 自动停止 · 新流到来 → start() 全速排空旧队列
单例守护线程永续运行，FIFO 队列保证行输出与字符输出严格有序

Context Chain

三模式上下文注入链

chat ↔ exec ↔ execs ↔ QQ 之间多向上下文同步，实现跨模式记忆穿透。统一记忆库架构确保AI不会失忆。

Chat 模式
ai/chat
← Agent结果注入 →
Agent 模式
ai/exec / ai/execs
↓ 上传
ConversationHistory
持久化 aiagent.db

① Chat → Agent

handleExec() 调用 agent.setChatHistoryContext(buildChatContextForAgent())， 取最近 6 条 Chat 消息（每条截断 200 字符）构建摘要，注入到 Agent 的 system prompt 的 "Recent Chat History" 段。

② Agent → Chat

Agent 执行完成后，getLastSummary() 获取本轮操作摘要， 在 Chat 的 system prompt 中注入，Chat 模式可感知 Agent 做了什么。

③ Agent → 下一轮 Agent（层层递进）

每轮 Agent 结束后，handleExec() 调用 agent.setPreviousSessionSummary(lastSummary)， 将本轮总结注入下一轮的 system prompt 的 "Previous Agent Session" 段。 配合 JournalManager 的跨会话日志，形成完整记忆链。

④ 辅助上下文注入口

MemoryManager：FTS5 全文搜索检索相关记忆 → 注入 system prompt
UnifiedQQMemoryManager：统一记忆库，所有对话存储在 unified_memory.db，通过 source_type/source_id 标记区分群聊/私聊，实现记忆联通
JournalManager：最近 25 条会话日志（跨重启持久化，SQLite）→ 注入 system prompt
EmotionManager：当前情绪状态（心情+性别，持久化到 app_state）→ 注入 system prompt
PersistenceManager：SQLite 统一持久化层，WAL 模式增量写入，替代 5 个 JSON 文件
UnifiedQQMemoryManager：QQ机器人统一记忆库，存储所有对话历史、群聊记录、AI独立记忆，通过来源标记实现记忆联通
以上所有上下文在 buildSystemPrompt() 中一次性融合。
